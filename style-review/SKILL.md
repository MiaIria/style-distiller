# /style-review — 档案审视

> 看你的"文字人格"全貌、变化轨迹、下一步该补什么。

---

## 触发

当用户说 `/style-review`、或表达"看看我的风格"、"档案全貌"、"我现在是什么风格"时，路由到这里。

---

## 0️⃣ 第一步：先读协同规则

```
Read ~/.claude/skills/style-lib/prompts/coordinate.md
```

了解 style-review 与其他 4 个 skill（feed/write/reject/feedback）的协同触发条件。

## 1️⃣ 第二步：读主流程

```
Read ~/.claude/skills/style-lib/prompts/review.md
```

按 review.md 中的流程完整执行。

## 🤝 第三步：协同工作（强制）

执行完主流程后，**必须**根据 coordinate.md 的"入口 E：style-review"触发矩阵：

| 触发条件 | 必须动作 |
|---------|---------|
| 状态 🔴 冷启动 | 提示"先 /style-feed 3-5 篇" |
| 状态 🟠 萌芽 | 末尾建议"继续 /style-feed 到 10 篇" |
| 状态 🟡 学习 | 末尾建议"试 /style-write 看实际效果" |
| 状态 🟢 成熟 | 末尾建议"持续使用 + 定期 review" |
| 某维度 ▱ > 5 | 提示"/style-feed 时重点标该维度" |
| 反样本 < 3 | 提示"/style-reject 标几个反例" |
| 档案 30 天未审视 | Append history.md 月度审视 |
| 用户要求导出 | 生成 export 文件 |

**反馈末尾必须带**：
```
💡 下一步建议（基于档案状态智能推荐）：
  - [A] {基于当前状态的最优下一步}
  - [B] {第二推荐}
  - [C] {第三推荐}
```

---

## 第二步：与用户的对话模板

`/style-review` 不需要任何参数，直接展示全貌。

如果有其他命令的反馈被路由到这里（用户表达"看看整体"），也直接进入审视流程。

---
