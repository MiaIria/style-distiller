# Style Distiller · 写作风格蒸馏器

> 一个会持续进化的"个人写作 AI"。
> 通过你日常喂入的好文 + 你自己写的文章，蒸馏出你的"文字人格"，
> 然后在你需要时按这个风格写出属于你的短文。

---

## 🎯 这是什么

你可能用过：
- 文心一言 / Kimi / ChatGPT 帮你"写文章" → 但写出来总不像你
- 各种风格模板（小红书体、知乎体）→ 但你不想变成一个模板

**Style Distiller 不替你写，它学你怎么写。**

它维护一份"你的文字人格档案"，每次你：
- 刷到好文 `/style-feed`
- 标出"这篇我讨厌" `/style-reject`
- 让它写一篇 `/style-write`
- 对生成结果打分 `/style-feedback`

档案就更新一次，你的 AI 就更懂你一点。

## 🚀 快速开始（5 分钟）

### 第一次使用

直接说：
```
/style-write 主题: 周末不想起床 长度: 300 平台: 小红书
```

系统会检测到档案为空，进入**冷启动引导**：
- 3 个问题（1 分钟）
- 1 次试写（30 秒）
- 完成！

### 喂入第一篇自我样本

```
/style-feed
[粘贴你自己写的一篇短文]
维度: 综合
备注: 这篇是我最近最满意的一篇
```

1 篇自我样本 = 3 篇外部样本，状态会从 🔴 → 🟠。

### 喂入第一篇外部样本

```
/style-feed
[粘贴你喜欢的一篇短文]
维度: 开头
备注: "开头那个反问直接把我钉住了"
```

### 解锁完整能力

继续 /style-feed 累计到 10 篇 → 🟡
累计到 30 篇 → 🟢（成熟）

## 📚 5 个核心指令

| 指令 | 何时用 | 输入 |
|------|--------|------|
| `/style-feed` | 刷到喜欢的文章 | 原文 + 维度 + 备注 |
| `/style-reject` | 看到讨厌的文章 | 原文 + 反感原因 |
| `/style-write` | 想写文章 | 主题 + 长度 + 平台 + 调性 |
| `/style-feedback` | AI 写完了 | 草稿 ID + 评分 + 采纳/不满意 |
| `/style-review` | 想看档案全貌 | （无） |

```
├── samples/
│   ├── positive/                   ← 你喜欢的文章
│   └── negative/                   ← 你反感的文章
├── drafts/                         ← AI 生成的草稿
├── backups/                        ← 最近 5 个档案备份
├── weights.json                    ← 维度权重 + 时间衰减
└── history.md                      ← 进化日志
```

**隐私**：所有数据只存在本地，不上传。

## 🧠 工作原理（简版）

Style Distiller 不能真改模型参数，它用 4 大机制模拟"训练"：

```
┌────────────────────────────────────────────┐
│  1. 上下文工程：把档案塞进每次生成的 prompt  │
│  2. 检索增强：从样本库找最相关的参考         │
│  3. 规则蒸馏：把样本特征翻译成"动作级"指令   │
│  4. 反馈循环：你的打分反向更新档案权重       │
└────────────────────────────────────────────┘
```

核心是"动作级提取"——不是"文笔细腻"这种空话，而是"开头必带时间锚定"这种 AI 能直接执行的具体动作。

详见 `prompts/extract.md`。

## 🎯 7 个风格维度

| 维度 | 问什么 | 文件 |
|------|--------|------|
| **钩子 hook** | 开头 1-2 句怎么抓人 | `profile/hook.md` |
| **节奏 rhythm** | 句长、断行、停顿 | `profile/rhythm.md` |
| **口气 voice** | 像什么人在说话 | `profile/voice.md` |
| **金句 verve** | 喜欢什么样的"亮句" | `profile/verve.md` |
| **收尾 closing** | 怎么结束有回味 | `profile/closing.md` |
| **词汇 vocabulary** | 常用词 / 禁用词 | `profile/vocabulary.md` |
| **格式 format** | 排版 / emoji / 标点 | `profile/format.md` |
| **人格 persona** | 综合画像 | `profile/persona.md` |

## 💡 写作场景（目前最佳）

基于设计，重点优化了**短文场景**：
- 小红书（300-800 字）
- 即刻（300-800 字）
- 朋友圈随笔（100-300 字）
- 微博（100-300 字）

长文场景（1500+ 字）会工作但不是最优。后续会扩展。

## 🛠 高级：直接编辑档案

所有档案是 Markdown 文件，可以手动编辑：

```bash
# 查看你的档案
code ~/.claude/styles/profile/persona.md

# 直接修改后，下次 /style-write 会用最新版本
```

**什么时候适合手改**：
- 觉得 AI 提的某个词你不常用 → 直接从 vocabulary.md 删掉
- 觉得档案画得不像你 → 直接改 persona.md
- 想要"近期偏好锋利一点" → 在 voice.md 顶部加一行

## 📈 进化路径建议

### 第 1 周：种子期
- 冷启动引导（3 分钟）
- /style-feed 2-3 篇你最满意的自我文章
- 试 /style-write 1-2 篇，看效果

### 第 1 个月：积累期
- 每天刷到好文顺手 /style-feed（10-15 篇）
- 偶尔 /style-reject 标记你不喜欢的
- 累计到 10-20 个正样本

### 第 3 个月：稳定期
- 累计 30+ 样本
- /style-write 几乎都"像你"
- 开始用 /style-feedback 微调

### 第 6 个月：自动化期
- 档案稳定，可以批量生成
- 每月 /style-review 一次看变化
- 开始尝试"反风格"生成（看看自己还能写什么）

## ❓ FAQ

**Q：会被 AI 写得像模板吗？**
A：不会。我们故意避免了"排比 + 升华 + 喊口号"这类模板化写法，反而偏好反类型。

**Q：我的档案会被别人看到吗？**
A：不会。所有数据在本地 `~/.claude/styles/`，不上传。

**Q：可以多人共享一份档案吗？**
A：可以。把 `~/.claude/styles/` 整个文件夹复制给朋友，他会得到一份"你的风格"，但他自己的档案独立演化。

**Q：可以重置档案吗？**
A：可以。`/style-review` → "重新初始化" 会清空所有数据并重新初始化。

**Q：和 create-ex 的关系？**
A：create-ex 蒸馏"一个前任"（人物 + 记忆），style-distiller 蒸馏"一种风格"（写法 + 偏好）。可以共存。

**Q：能蒸馏其他语言吗？**
A：当前设计针对**简体中文短文**。其他语言可用但没优化。
