# 风格生成验证（最关键的质量保证）

> **目的**：生成完不直接给用户，先让 LLM 自己核对档案的硬约束。
> 这是"训练 50 篇不白费"的核心保障。

---

## 为什么需要这一步

LLM 有强烈的训练数据偏置——它默认会写出"小红书体"、"鸡汤体"、"治愈体"。

**就算你档案里写"克制冷静"，LLM 也可能一不小心冒出"在这个浮躁的时代"这种话。**

所以必须**生成后自动核对**，不能相信 LLM 的"自觉"。

---

## 验证流程（在 write.md 流程中嵌入）

### Step V1：7 维硬约束核对

逐项检查生成结果：

| 维度 | 检查内容 | 判定方法 |
|------|---------|---------|
| **钩子** | 开头 1-2 句类型 | 匹配 profile/hook.md 的稳定类型 |
| **节奏** | 句长分布 | 计算平均句长、短句占比 |
| **口气** | 主调 + 温度 | 检测禁用词 / 禁用句式 |
| **金句** | 至少 1 个符合偏好 | 检测金句类型 |
| **收尾** | 收尾方式 | 匹配 closing.md |
| **词汇** | 无禁用词 | 扫描 vocabulary.md 忌讳词清单 |
| **格式** | 分段 / emoji / 标点 | 匹配 format.md |

**每项输出**：✅ 完全符合 / ⚠️ 基本符合 / ❌ 严重偏离

### Step V2：反样本规避扫描

```python
# 伪代码
for word in vocabulary.md.忌讳词:
    if word in 草稿:
        标记 "❌ 用了禁用词：{word}"

for pattern in persona.md.禁忌区:
    if pattern.出现 in 草稿:
        标记 "❌ 触发禁忌：{pattern}"
```

**输出**：100% 规避 / 部分规避 / 完全没规避

### Step V3：样本相关性核对

检查 prompt 里引用的 3-5 个样本，**这些样本的特征是否真的出现在生成结果里**：

| 引用样本 | 应该用上的特征 | 实际是否用上 |
|---------|--------------|------------|
| #005 | 场景开场 + 时间锚定 | ✅ 是 |
| #012 | 自嘲 + 短句推进 | ⚠️ 部分 |
| #023 | 留白收尾 | ❌ 没看到 |

如果"应该用上"的核心特征**实际用上 < 50%**，**视为风格没还原**。

### Step V4：主题完成度

检查：
- ✅ 主题是否写到了？
- ✅ 是否偏题/跑题？
- ✅ 字数是否达标？
- ✅ 平台调性是否对？

### Step V5：综合偏离度评分

```
偏离度 = (
  7 维硬约束违反数 × 0.5
  + 反样本触发数 × 0.3
  + 样本特征未还原比例 × 0.2
) × 100
```

| 偏离度 | 等级 | 处理方式 |
|--------|------|---------|
| 0-10% | 🟢 高度还原 | 直接给用户 |
| 10-25% | 🟡 较好 | 自动修正（改写偏离的句子） |
| 25-50% | 🟠 部分偏离 | 标注问题段，建议重写 |
| > 50% | 🔴 严重偏离 | **必须重写**（自动跑 1 次） |

---

## 自动修正流程

当偏离度 10-25% 时，自动修复：

```markdown
检测到以下偏离：

❌ 用了禁用词："在这个浮躁的时代"（第 3 段）
❌ 偏离口气：第 5 段偏向"鸡汤"（"愿你..."）
❌ 收尾不符：最后一句升华型，档案偏好留白

自动修正后：

{修正后的草稿}
```

**修正规则**：
- 禁用词 → 直接替换为档案里的偏好词
- 口气偏离 → 改写该句
- 收尾不符 → 重写最后 1-2 句

修正后再跑一次 V1-V4，确认偏离度 < 10% 才给用户。

---

## 自动重写流程

当偏离度 > 50% 时：

1. **不直接给用户当前草稿**
2. **分析失败原因**：
   - 是 prompt 组装没传达到？
   - 是 LLM 偏置太强？
   - 是档案本身有矛盾？
3. **重新组装 prompt**（加强约束：把档案原文嵌入而不是摘要）
4. **再次生成**
5. **再次验证**
6. **最多重试 2 次**，如果还不行，**告诉用户"档案需要补充"**

## 失败兜底

万一 2 次重试后还是不行，**绝对不能装作没事给用户**，必须明确告知：

```
⚠️ 风格还原度低，建议补充档案

我尝试了 2 次重写，但生成结果仍然偏离你的档案。

可能原因：
  1. 档案覆盖度不够（你目前 {N} 个样本，建议 30+）
  2. 该主题档案空白（建议 /style-feed 几篇相关主题）
  3. 档案内部有冲突（建议 /style-review 检查）

下一步建议：
  - [A] /style-feed 几篇类似主题的文章
  - [B] /style-review 检查档案一致性
  - [C] 先用当前结果，后续补充档案后再优化
```

## 验证 vs 写后用户反馈的关系

| 阶段 | 验证 | 反馈 |
|------|------|------|
| 时机 | 生成后立即 | 用户看完后 |
| 触发者 | 系统自动 | 用户主动 |
| 作用 | 防"档案白费" | 反向校准档案 |
| 失败处理 | 重写或告知 | 写入 history.md |

两者**协同工作**：
- 验证保证"这次生成不白费训练"
- 反馈保证"下次生成更准"
