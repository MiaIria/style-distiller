# /style-feedback 流程（反馈回流）

> **目的**：对生成的草稿打分/批注，反向校准档案，让下次生成更准。

## 触发后第一件事

询问用户反馈对象：
- 草稿 ID：{如 drafts/2026-06-02_001_xxx.md} 或 "最近生成的那篇"
- 如果没有草稿（用户想对正/反样本做反馈）→ 询问是哪种反馈：
  - [A] 对生成草稿的反馈（最常见）
  - [B] 对档案本身的纠错（"其实我不常用 X 词"）
  - [C] 对样本的纠错（"那个样本其实没那么喜欢"）

## 主流程（场景 A：对生成草稿的反馈）

### Step 1：读取草稿和它的风格报告

`Read` 草稿文件（包括元数据、原文、风格报告）。

### Step 2：解析用户反馈

可能的反馈形式：

```
[评分]
  - 1-10 分
  - 或更粗：👍 / 👎 / 😐

[采纳部分]
  - 开头 / 中段 / 结尾
  - 某一句话（用户可能贴出来）
  - 某个特征："那个钩子我喜欢"

[不满意部分]
  - 同样的形式
  - 用户原文："开头太套路了"

[附加说明]（可选）
  - 自由文本
```

如果用户只给了评分（1-10），自动归类：
- 1-3 分 → 大部分不满意
- 4-6 分 → 部分采纳
- 7-8 分 → 大部分采纳
- 9-10 分 → 几乎完美，作为"标杆样本"

### Step 3：校准档案

#### 3.1 采纳的特征 → 强化

找到"采纳部分"对应的特征，**在对应 profile 里 +0.1 权重**（在文本上用"⭐ 强化"标注）：

```markdown
## 钩子类型分布

| 类型 | 占比 | 强度 |
|------|------|------|
| 场景开场 | 40% | ⭐ 强化（来自草稿 #001 反馈）|
| 反问开头 | 25% | |
```

#### 3.2 不满意的特征 → 弱化 / 写入反特征

找到"不满意部分"对应的特征：
- 在对应 profile 里 -0.1 权重（用"⚠️ 弱化"标注）
- 如果是反复出现的问题 → 升为"反特征"
- 如果是新增问题 → 加入"待观察"清单

#### 3.3 评分极低（1-3）→ 触发"风格回退"

如果单次评分 ≤ 3：
- 自动备份当前档案
- 把这次反馈记录到 `history.md` 的"⚠️ 风格回退"区
- 提示用户是否要回滚

### Step 4：写入 history.md

```markdown
## {日期}

### 草稿 #001 反馈（生成于 {生成时间}）
- 评分：7/10
- 采纳：开头场景、整体节奏
- 不满意：结尾太散了
- 档案变化：
  - hook.md：场景开场 +0.1
  - closing.md：留白型 -0.1
  - 标记：用户偏好"回扣型"或"短句炸点型"收尾
- 进化建议：补 1-2 篇用户喜欢的收尾样本
```

### Step 5：反馈给用户

```
📊 反馈已记录！

📝 草稿 #001（生成于 {时间}）
⭐ 评分：7/10

✅ 采纳：开头场景、整体节奏
❌ 不满意：结尾太散

📈 档案变化：
  - hook：场景开场 +0.1
  - closing：留白型 -0.1 → 标记为"待观察"

💡 建议：
  - 下次补 1-2 篇你喜欢的收尾样本
  - /style-review 可看完整变化曲线

下次 /style-write 会更准。
```

## 场景 B：对档案本身的纠错

用户可能说：
- "其实我不用'其实'这个词" → 从 vocabulary.md 的"常用口头禅"中移除
- "我不是克制，我是温和" → 改 voice.md 的主调
- "我最近想试锋利一点的" → 在 voice.md 增加"近期偏好"区

处理：
1. `Read` 涉及到的 profile
2. `Edit` 修正
3. 写入 `history.md` 的"档案手动纠错"区
4. 反馈给用户

## 场景 C：对样本的纠错

用户可能说：
- "那个样本其实没那么好" → 标记为"低权重"或"撤回"
- "那个样本的标题我喜欢但正文一般" → 拆分或备注

处理：
1. `Read` 样本文件
2. 调整 weight 字段或备注
3. 如撤回：删除 + 差分回滚
4. 写入 `history.md`

---

## 校准权重的规则

| 操作 | 权重变化 |
|------|---------|
| 用户采纳 | +0.1（单个特征） |
| 用户不满意 | -0.1 |
| 用户极不满意（1-3 分） | -0.3 + 备份 |
| 用户纠错档案 | 直接修改，权重 = 1.0 重新计 |
| 时间衰减 | 每月 -0.05 |

权重封顶 1.5（防止单一特征过度主导），下限 0.3（保留但弱化）。

---

## 边界情况

### 1. 用户评分 10/10
- 触发"标杆样本"：把这次生成的反向归档到 `samples/positive/`，作为 AI 自己生成的"我喜欢的文章"
- 这是非常强的"自学习"信号

### 2. 用户对档案的纠错和样本冲突
- 提示："你之前喂的 #005 样本里常用'其实'，但你现在说你不用。要处理吗？"
  - [A] 样本撤回
  - [B] 样本保留但降低权重
  - [C] 你现在变了，保留样本作为历史

### 3. 用户不评分只说"还行"或"凑合"
- 默认 6 分，归类"部分采纳"——意味着档案命中但未完美
