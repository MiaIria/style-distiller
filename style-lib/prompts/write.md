# /style-write 流程（按你的风格生成文章）

> **目的**：加载最新档案 + 检索相关样本 → 组装风格 prompt → 生成草稿 + 风格报告。

## 触发后第一件事

读 `~/.claude/styles/weights.json` 判断状态：
- 🔴 冷启动（0-2 样本）→ 转 `${CLAUDE_SKILL_DIR}/prompts/cold_start.md`
- 🟠 萌芽（3-9）→ 继续，但风格置信度低
- 🟡/🟢 → 正常流程

## 解析用户输入

识别以下信息：

```
[主题] —— 必填
  - 写什么：可以是关键词 / 一句话 / 一个问题

[长度] —— 必填
  - 短（100-300 字）：朋友圈/微博随笔
  - 中（300-800 字）：小红书/即刻
  - 长（800-1500 字）：偏长的推文
  - 或指定具体字数

[平台] —— 必填
  - 朋友圈
  - 微博
  - 小红书
  - 即刻
  - 公众号（短文）
  - 其他

[调性] —— 可选
  - 默认按 voice 档案
  - 用户可指定：克制 / 锋利 / 温柔 / 冷幽默 / 自嘲
  - 或反向："不要太丧" / "振奋一点" / "温和一些"

[特殊要求] —— 可选
  - "加一个具体场景" / "结尾要留白" / "开头要反问"
```

如果缺少关键信息（主题/长度/平台），用一次 AskUserQuestion 问清楚。

## 主流程

### Step 1：加载 8 份 profile

并行 `Read` 8 个文件：
- `~/.claude/styles/profile/hook.md`
- `~/.claude/styles/profile/rhythm.md`
- `~/.claude/styles/profile/voice.md`
- `~/.claude/styles/profile/verve.md`
- `~/.claude/styles/profile/closing.md`
- `~/.claude/styles/profile/vocabulary.md`
- `~/.claude/styles/profile/format.md`
- `~/.claude/styles/profile/persona.md`

### Step 2：检索最相关的 3-5 个样本

> ⚠️ **这一步决定训练会不会白费**。挑错样本 = LLM 学不到典型特征 = 风格还原失败。

**严格按 `~/.claude/skills/style-lib/prompts/retrieval.md` 的多路召回策略执行**。

`Bash` 列出所有正样本：
```bash
ls ~/.claude/styles/samples/positive/
```

对每个候选样本：
1. `Read` 它的 frontmatter 提取主题、维度、备注
2. 计算综合分（主题 0.5 + 维度 0.3 + 时效 0.2）
3. 挑出 top 3-5

**生成检索报告**（让用户看到挑了哪些）：
```
🔍 检索报告
  1. #012 - 0.92 (主题 0.95 + 维度 0.80 + 时效 0.95)
  2. #023 - 0.87 (主题 0.90 + 维度 0.75 + 时效 0.90)
  3. #005 - 0.83 (主题 0.85 + 维度 0.85 + 时效 0.78)
  主题相关样本：3/50（覆盖度 6%，建议补样本）
```

**关键样本的特征提取**（用于后续验证）：
对每个挑中的样本，提取 7 维特征签名（hook 类型、节奏、口气、金句、收尾、词汇、格式），后续生成后逐项核对。

### Step 3：检查反样本

`Bash` 列出反样本：
```bash
ls -la ~/.claude/styles/samples/negative/ | head -20
```

如果存在反样本，`Read` 它们的反特征清单，**组装到 prompt 时主动避开**。

### Step 4：组装风格 prompt（分层硬约束）

> ⚠️ **这是另一个决定训练白不白费的关键节点**。档案翻译成"软描述"等于白搭。

**Prompt 必须分层**——核心约束用 🚨 标记 + 重复，软约束给范围，参考样本嵌入原文。

```markdown
# ⚠️ 任务（高优先级）

你是一个**风格严格的写作者**。本次任务的"风格"权重 > "内容"权重。
如果风格不达标，内容再好也不算成功。

用我提供的"风格档案"，写一篇关于「{主题}」的{长度}字文章，用于{平台}。
{调性要求}

═══════════════════════════════════════
# 🚨 硬约束（绝对不能违反）
═══════════════════════════════════════

## 🚫 禁用词（出现任何一个 = 失败）
{vocabulary.md 忌讳词清单}

## 🚫 禁用句式（出现 = 失败）
{persona.md 禁忌区}

═══════════════════════════════════════
# 🟡 强约束（必须遵守）
═══════════════════════════════════════

## 钩子（开头 1-2 句）
- 类型：{hook.md 中占比 ≥ 30% 的类型}
- 动作：{hook.md 的"动作级偏好"前 3 条}

## 口气
- 主调：{voice.md 的"主体调性"}
- 温度/锋利度：{voice.md 的具体位置}
- 偏好人称：{voice.md 的"自指频率"}

## 收尾
- 偏好类型：{closing.md 中占比 ≥ 30% 的类型}
- 收尾前 1-2 句：{closing.md 的"收尾前准备"}

═══════════════════════════════════════
# 🟢 软约束（尽量遵守）
═══════════════════════════════════════

## 节奏
- 平均句长：{rhythm.md 的"基础指标"平均值} 字（±20% 可接受）
- 短句占比：{rhythm.md 的短句占比}%
- 断点句：{rhythm.md 的"断点句习惯"}

## 金句
- 偏好类型：{verve.md 中占比 ≥ 30% 的类型}
- 位置：{verve.md 的"金句位置偏好"}
- 意象库：{vocabulary.md 的"偏好意象"}

## 词汇偏好
- 常用词：{vocabulary.md 的"常用口头禅"}
- 偏好动词：{vocabulary.md 的"偏好动词"}

## 格式
- 段落：{format.md 的段落偏好}
- emoji：{format.md 的 emoji 密度}
- 标点：{format.md 的标点偏好}

═══════════════════════════════════════
# 📚 参考样本（重点用上其特征）
═══════════════════════════════════════

> 下方 3-5 个样本的特征**必须**在生成结果中体现至少 80%。
> 生成后我会用 verify.md 流程核对是否真的用上了。

## 样本 A：{id} - {主题}（{维度}）
[完整原文]
**特征签名**：
  - 钩子：场景开场+时间锚定
  - 节奏：短句 8 个 + 长句 1 个
  - 口气：克制冷静
  - 金句：反转型（"原来不是世界变安静了"）
  - 收尾：留白

## 样本 B：{id} - {主题}（{维度}）
[完整原文]
**特征签名**：...

═══════════════════════════════════════
# 📝 主题素材
═══════════════════════════════════════

{用户提供的主题信息 / 上下文}

═══════════════════════════════════════
# ✅ 输出要求
═══════════════════════════════════════

1. 严格遵守硬约束（禁用词/句式/反特征）
2. 满足强约束（钩子/口气/收尾）
3. 尽量符合软约束（节奏/金句/词汇/格式）
4. 用上至少 3 个样本的核心特征
5. 字数 {目标} ±20%
6. 输出文章正文 + 自报家门（用了哪些样本特征）
```

### Step 5：生成草稿

将组装好的 prompt 作为 LLM 上下文的一部分，生成草稿。

**生成时的特殊指令**：
- 让 LLM 在生成时**显式标记**用了哪些档案特征（自报家门）
- 让 LLM 给出 3 个微调方向
- 不要过度使用档案里的特征——风格是"用得自然"，不是"处处用"

### Step 5.5：⚠️ 强制质量验证（不能跳过）

> **LLM 容易"嘴硬"自报做对了，必须独立验证**。

**严格按 `~/.claude/skills/style-lib/prompts/verify.md` 流程执行**：

1. **7 维硬约束核对**（钩子/节奏/口气/金句/收尾/词汇/格式）
2. **反样本规避扫描**（扫描所有禁用词/禁用句式）
3. **样本特征还原核对**（挑中的样本特征是否真的用上了）
4. **主题完成度检查**
5. **计算偏离度评分**

**根据偏离度处理**：
- 0-10%：🟢 直接进入 Step 6
- 10-25%：🟡 自动修正后进入 Step 6
- 25-50%：🟠 标注问题段，建议重写
- > 50%：🔴 **必须重写**（最多 2 次），仍不行则告知用户"档案需要补充"

### Step 6：后处理

#### 6.1 字数校准
检查字数：
- 短文 < 80 字 → "太短了，要不要扩写？"
- 短文 > 200 字 → "超出预期，要不要精简？"

#### 6.2 风格违规检查
逐项检查 prompt 里的"避免"清单是否被违反：
- ❌ 用了禁用词？→ 提醒用户并提供替换
- ❌ 用了反样本的写法？→ 重写该句

#### 6.3 自动存档草稿（含质量报告）
把生成的草稿写入 `~/.claude/styles/drafts/{YYYY-MM-DD}_{NNN}_{slug}.md`：

```markdown
---
id: 2026-06-02_001
generated_at: 2026-06-02
topic: "..."
length: "短"
platform: "小红书"
samples_used: ["2026-06-02_001", "2026-06-01_002"]
samples_count_used: 3
deviation_score: 0.08
verification: passed  ← passed / auto_fixed / rewrite_required
---

# 草稿

[生成的文章正文]

---

# 📊 风格匹配度报告（来自 verify.md）

**综合匹配度：92% 🟢**

| 维度 | 状态 | 说明 |
|------|------|------|
| 钩子 | ✅ | 场景开场（档案偏好 35%，本稿使用）|
| 节奏 | ✅ | 短句 55%（档案基准 50%，达标）|
| 口气 | ✅ | 克制冷静，无禁用词 |
| 金句 | ✅ | 1 个反转型（档案偏好）|
| 收尾 | ✅ | 留白（档案偏好）|
| 词汇 | ✅ | 未触发任何忌讳词 |
| 格式 | ✅ | 分段合理，无 emoji 堆砌 |

**样本参考核对**：
- #005 场景开场 ✓ 用上了
- #012 自嘲口气 ✓ 用上了
- #023 留白收尾 ✓ 用上了
- #038 具象金句 ✓ 用上了

**反样本规避**：100%（9 条忌讳词未触发）

**主题完成度**：95%
- 主题"35 岁感悟"已覆盖
- 字数：412 字（目标 400 ± 20%）
- 平台：符合小红书调性

# 微调方向

[1] ...
[2] ...
[3] ...
```

### Step 7：呈现给用户

```
📝 草稿已生成（保存到 drafts/2026-06-02_001_xxx.md）

{草稿正文}

────────────────────
📊 风格报告（置信度：🟡 中等）
  ✓ 钩子用了 [场景开场]，参考样本 #001
  ✓ 节奏匹配：短句 55%
  ✓ 口气：克制冷静
  ✓ 收尾：[留白]
  ⚠️ 反样本检查：全部避开

🎯 微调方向（挑一个让我改）：
  [1] 开头再炸一点（用反问替代场景）
  [2] 结尾换成 [回扣] 而不是 [留白]
  [3] 整体再克制 30%（去掉一个金句）

满意的话 /style-feedback 打分，让我记住这次的偏好。
不满意的话直接说"哪里不对"，我重写。
```

## 风格置信度算法

```
置信度 = (
  总样本量 / 30 × 0.4
  + 弱维度样本数 / 10 × 0.3
  + 主题匹配度 × 0.2
  + 反样本数 × 0.1
) capped at 1.0
```

可视化：
- 0.0-0.2：🔴 几乎通用
- 0.2-0.5：🟠 有点感觉
- 0.5-0.8：🟡 比较像
- 0.8-1.0：🟢 很像你

## 边界情况

### 1. 用户没给主题
- 询问："想写什么主题？给我 1 句话、几个关键词、或者一个你想探讨的问题都行。"

### 2. 用户给的主题档案里没相关样本
- 用通用生成 + 强提示"档案里没有这个主题的样本，生成结果会偏通用"
- 建议用户："要不要先 /style-feed 喂一篇类似主题的文章？"

### 3. 用户连续 /style-write 5 次主题都不同
- 提醒："你的档案在不同主题上覆盖不均，要不要 /style-review 看看该补哪个？"

### 4. 用户要求改稿
- 把"改稿"路由到 `/style-feedback`（如果是对自己写的改）
- 或把"改稿"路由到 `/style-write` 的变体（如果是改 AI 生成的）

### 5. 用户想要"反风格"——故意不写自己
- 在主题里加 `{调性: 反风格 / 故意不像我}`
- 生成时弱化档案约束，只取硬禁忌（反样本）规避
