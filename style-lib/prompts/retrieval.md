# 样本检索策略（防止"挑错样本"导致档案白费）

> **目的**：从 N 个样本里挑出 3-5 个最相关的，喂给 LLM。
> 挑错样本 = LLM 看不到典型特征 = 训练白费。

---

## 检索的 3 大风险

### 风险 1：主题相似度低
- 用户写"35 岁感悟"，档案里有 30 篇"职场吐槽"
- 挑到"职场吐槽" → LLM 学不到"35 岁"主题的典型表达

### 风险 2：维度匹配度低
- 用户偏好"克制冷静"，挑到 5 篇"锋利犀利"
- LLM 写出来可能更锋利

### 风险 3：样本过时
- 档案里最早的几篇是用户 1 年前训练的风格
- 用户风格已经变了
- LLM 学的是"过去的你"

---

## 多路召回 + 相关性排序

### 路 1：主题关键词重合

```python
def topic_score(sample, user_topic):
    sample_keywords = extract_keywords(sample.content)
    topic_keywords = extract_keywords(user_topic)
    overlap = len(sample_keywords & topic_keywords)
    return overlap / max(len(topic_keywords), 1)
```

权重 0.5

### 路 2：维度匹配度

```python
def dimension_score(sample, user_preference):
    matched_dims = set(sample.frontmatter.dimensions) & set(user_preference)
    return len(matched_dims) / max(len(user_preference), 1)
```

权重 0.3

### 路 3：时效权重

```python
def time_score(sample):
    months_old = (now - sample.fed_at).days / 30
    return 0.95 ** months_old  # 每月衰减 5%
```

权重 0.2

### 综合评分

```
final_score = topic_score * 0.5 + dimension_score * 0.3 + time_score * 0.2
```

按 final_score 排序，取前 3-5 个。

## 兜底：样本不足时怎么办

### 情况 1：相关样本 < 3
- 用通用能力 + 档案硬约束
- 显式告知用户"该主题档案不足"

### 情况 2：相关样本 = 0
- 用通用能力 + 标注"档案里没有该主题样本"
- 建议用户先 /style-feed

### 情况 3：所有样本都不相关
- 这通常意味着档案覆盖窄
- 强烈建议 /style-feed 补相关主题

## 样本特征签名（防"挑错"的进阶机制）

为了避免"挑到一篇但 LLM 没学到关键特征"，**给每个样本提取特征签名**：

```yaml
# 样本 #005 的特征签名
signature:
  hook: "场景开场+时间锚定（'周一上午9点'）"
  rhythm: "短句 8 个 + 长句 1 个"
  voice: "克制冷静"
  verve: "反转型（'原来不是世界变安静了'）"
  closing: "留白"
  vocabulary: ["其实", "你看", "听"]
  distinctive: "用动作表达情绪而非形容词"
```

LLM 生成时，**对比草稿 vs 样本特征签名**，看是否真的用上了。

---

## 防止"风格被压过"的检索策略

**问题**：档案里 50 篇，LLM 默认会写"小红书体"（训练数据偏置）

**解决**：
1. **样本中选 1-2 个"反小红书体"**的样本（如果用户档案有）
2. **把这些样本的特征签名做"高亮"**：
   ```markdown
   ⚠️ 特别注意：以下 2 个样本的特征是你必须用上的，否则会被小红书体压过：
   - #005：场景开场+感官细节
   - #012：克制+留白（不要升华）
   ```
3. **重复 2-3 次强调**（LLM 注意力分散，要靠重复）

---

## 检索与档案新鲜度的关系

档案会随时间变"旧"，需要定期重新计算样本权重：

```
权重 = 1.0 × 0.95 ^ 月龄
```

- 1 个月前：0.95
- 3 个月前：0.86
- 6 个月前：0.74
- 1 年前：0.54

**注意**：检索时按 fresh_score（含时间衰减）排序，而不是单纯按主题匹配度。

## 检索报告（让用户知道挑了哪些）

每次 /style-write 时，给用户看：

```
🔍 检索报告

主题"35 岁感悟"相关样本排序：
  1. #012 - 综合分 0.92 (主题 0.95 + 维度 0.80 + 时效 0.95)
  2. #023 - 综合分 0.87 (主题 0.90 + 维度 0.75 + 时效 0.90)
  3. #005 - 综合分 0.83 (主题 0.85 + 维度 0.85 + 时效 0.78)
  ...

挑选了前 3 个：
  - #012：场景开场+自嘲口气（你最相关主题）
  - #023：留白收尾+克制
  - #005：具象金句（你早期的标杆）

⚠️ 提示：档案里"35 岁感悟"主题样本较少（3/50），
   建议 /style-feed 几篇同主题文章
```

**用户看到就知道**：
- LLM 真的用了哪些样本
- 档案覆盖度如何
- 是否需要补样本

## 检索失败的兜底

如果 50 个样本里挑不到 1 个相关的：

```
🚨 检索失败：档案里没有该主题的样本

你的档案有 50 个样本，但没有一个主题与"量子物理入门"相关。

可能原因：
  - 你训练的 50 篇都集中在"职场"和"生活感悟"
  - 量子物理是全新主题

建议：
  1. /style-feed 2-3 篇量子物理相关的文章
  2. 或者换一个有档案覆盖的主题

是否仍然要用通用模式生成？（不保证风格像你）
```

---

## 关键原则

1. **宁缺毋滥**：样本不足时显式告知，不要硬撑
2. **可见性**：让用户看到检索过程，不黑盒
3. **持续优化**：检索报告给用户→用户决定是否补样本→下次更准
