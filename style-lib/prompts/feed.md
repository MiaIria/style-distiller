# /style-feed 流程（正样本喂入）

> **目的**：把用户喜欢的文章/段落喂入风格档案，蒸馏出可迁移的"动作级"风格特征。

## 触发后第一件事

读 `~/.claude/styles/weights.json` 判断当前状态：
- 🔴 冷启动（0-2 样本）→ 转 `${CLAUDE_SKILL_DIR}/prompts/cold_start.md`
- 🟠/🟡/🟢 → 进入主流程

## 主流程

### Step 1：解析用户输入

用户的输入可能包含以下部分（智能识别，不要硬问）：

```
[原文] —— 必填
  - 直接粘贴的文章/段落
  - 或 URL（用 WebFetch 抓取）
  - 或图片（用 Read 工具读 OCR 内容）
  - 或 [我自己写的] 标识

[维度] —— 可选
  - 内容 / 话题 / 表达方式 / 文字布局 / 综合
  - 多个维度用逗号分隔

[备注] —— 强烈建议
  - 为什么这篇文章吸引你
  - 一句话即可，例："开头那个反问直接把我钉住了"
  - 或："最后一句的留白让我想了一晚上"
```

如果用户输入不完整，**用一次 AskUserQuestion 追问**：
- 原文：❓ 没看到原文，能再贴一下吗？
- 维度：可以推断出大致方向（开头的钩子 / 整篇的口气 / 收尾方式...），**不要强迫用户选**
- 备注：可以追问"哪一句最让你有感触？"

> **原则**：让用户写"为什么喜欢"比填"维度标签"重要 10 倍。备注是金，维度是银。

### Step 2：分析样本（动作级提取）

按 `${CLAUDE_SKILL_DIR}/prompts/extract.md` 的方法论，对原文做 7 维提取。

**特别关注用户备注**：
- 备注里点出的句子/段落 → 在该维度上 + 详细分析
- 没提到的维度 → 简略提取或跳过

输出结构：
```markdown
## 样本提取

### 钩子
- 类型：场景开场 70% / 反问 30%
- 关键证据：原文第 1 句"周一上午 9 点..."
- 动作偏好：开头必带时间锚定

### 节奏
- 平均句长：18 字
- 短句占比：55%
- ...

### 口气
- 主调：克制冷静
- ...

### 金句
- 检测到 3 个候选金句
- "原来不是世界变安静了" ← 反转型
- ...

### 收尾
- 类型：留白
- ...

### 词汇
- 常见口头禅：其实、说白了
- 偏好意象：旧书、菜市场
- 忌讳词：（未检测到）

### 格式
- 段落短（平均 1.8 句）
- 无 emoji
- 偏好破折号
```

### Step 3：生成样本文件

文件名格式：`{YYYY-MM-DD}_{NNN}_{slug}.md`
- NNN = 3 位序号（按时间顺序）
- slug = 用户备注的前 10 个字（去标点、英文小写、连字符分隔）

示例：`2026-06-02_001_开头反问钉住我.md`

用 `Write` 写入 `~/.claude/styles/samples/positive/2026-06-02_001_开头反问钉住我.md`：

```markdown
---
id: 2026-06-02_001
source: 抖音/公众号/我自己的/...
fed_at: 2026-06-02
dimensions: [hook]
weight: 1.0
remark: "开头那个反问直接把我钉住了"
---

# 原文

[完整原文]

---

## 提取结果

### 钩子 hook
- 类型：反问开头 100%
- 关键证据：第 1 句"你有没有想过，为什么..."
- 动作：二选一式反问开场
- 反特征：（无）

### 节奏 rhythm
- 平均句长：19 字
- 短句占比：48%
- ...

[... 其他维度，简洁版]
```

### Step 4：差分更新 8 份 profile

逐个读取并 Edit 8 份 profile 文件：

#### profile/hook.md
- 找到"钩子类型分布"表格，在对应类型 +1 样本
- 在"动作级偏好"添加 1-2 条新发现的特征
- 在"典型案例"中追加一句（原文第 1 句 + 标注）

#### profile/rhythm.md
- 用新样本的指标更新"基础指标"
- "断点句习惯"追加新发现

#### profile/voice.md
- 更新"主体调性"的勾选状态
- 更新温度/锋利度位置
- "人格比喻"区域追加新观察

#### profile/verve.md
- "金句类型分布"表格 +1
- "典型金句集"追加 1-2 句
- 标注来源样本 ID

#### profile/closing.md
- "收尾类型分布"表格 +1
- "典型收尾集"追加 1 句

#### profile/vocabulary.md
- 累加"常用口头禅"
- 累加"偏好动词"
- 累加"偏好意象"
- 累加"领域术语"

#### profile/format.md
- 更新段落、空行、emoji、标点指标

#### profile/persona.md
- 更新"风格稳定度"表格
- 更新"总样本量"和"风格状态"
- 如果样本量跨过状态阈值（2→3, 9→10, 29→30），更新状态和人格画像

**Edit 时的"差分"原则**：
- 同一特征在 ≥ 60% 样本中 → "稳定特征"
- 30-60% → "变体特征"，标注占比
- < 30% → "偶发特征"，不主动模仿
- 冲突 → 触发"分流"提示（见 Step 6）

### Step 5：更新 weights.json

用 `Edit` 更新：
- `total_samples_positive` +1
- `last_updated` 改为今天
- `current_state` 根据样本量重新计算

### Step 6：冲突检测（可选但推荐）

如果新样本与现有档案在某维度**强冲突**（如档案说"偏好长句"，新样本是"通篇短句"），执行：

1. 在 `history.md` 记录冲突
2. 输出 `⚠️ 风格冲突提示` 给用户：
   ```
   ⚠️ 检测到风格冲突：
   
   现有档案"节奏"维度显示：偏好中长句（平均 28 字）
   本次样本"节奏"维度显示：通篇短句（平均 15 字）
   
   可能原因：
   [A] 样本是特例（如别人写的）→ 标记为"偶发"
   [B] 你的风格在演变 → 触发档案分流
   [C] 这篇文章在某个特定方面刻意不同（如讽刺文体）→ 单独标注
   
   建议处理方式？（直接告诉我或选）
   ```

### Step 7：反馈给用户

```
✅ 样本已吸收！

📝 ID：2026-06-02_001
📂 入库：samples/positive/
🎯 维度：hook / rhythm / voice

📊 档案变化：
  - hook 维度：场景开场 +1（占比从 30% → 35%）
  - 词汇：新增常用词"其实"、"你看"
  - 状态：🟠 萌芽（3 样本）

💡 进化建议：
  - 你还缺 verve（金句）维度的样本
  - 下次刷到喜欢的金句，标 verve 试试
```

如果状态跨级（冷启动→萌芽等），加一句提示：
```
🎉 档案状态从 🔴 冷启动 → 🟠 萌芽
   现在可以尝试 /style-write 了！可能还不够"像你"，但能看个雏形。
```

## 边界情况

### 1. 用户直接说"我喜欢这篇"但没贴文章
- 问"能贴一下原文吗？或者告诉我在哪儿能找到？"

### 2. 用户给了一篇 5000 字长文
- 先提醒："这是长文，跟我们短文场景不太匹配。挑 1-2 段你觉得最有感觉的段落喂给我？"

### 3. 用户给的样本是他自己写的
- 在 frontmatter 标 `source: self`
- 同样按 7 维提取，但权重默认 1.2（自我样本价值更高）
- 在 `persona.md` 加注"自我样本"

### 4. 用户连续喂了 5 篇同一个作者
- 触发提醒："你这 5 篇都是 X 的。我会把它们作为 X 的风格标记，但记住你的档案是'你喜欢的风格'，可能与你自己的风格不同。"

### 5. 用户撤回样本
- 删除 `samples/positive/{id}.md`
- 重新差分更新 8 份 profile（回滚该样本的贡献）
- 更新 `weights.json` 样本量
