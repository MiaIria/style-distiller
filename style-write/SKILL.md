# /style-write — 按你的风格写文章

> 加载你训练好的风格档案，按"你的写法"生成短文。
> **承诺：训练 50 篇不会白费**。生成后自动验证风格还原度，不达标自动重写。

---

## 🛡️ 5 道质量保证护城河

| # | 护城河 | 解决什么 |
|---|--------|---------|
| 1 | **检索要准** | 50 个样本里挑 3-5 个最相关的（按 `retrieval.md` 多路召回）|
| 2 | **档案翻译要硬** | 软描述 → 硬约束，分 🚨/🟡/🟢 三层 |
| 3 | **Prompt 组装要全** | 嵌入完整原文 + 特征签名，不只是摘要 |
| 4 | **生成后要自动验证** | 7 维核对 + 反样本扫描 + 样本特征还原核对 |
| 5 | **偏离要自动修复** | 偏离 10-25% 自动改，>50% 自动重写，最多 2 次 |

> ⚠️ **绝不接受"档案是档案，生成是生成"**。如果最后生成结果和档案无关，这是一次失败的任务。

## 🤝 第三步：协同工作（强制）

执行完主流程后，**必须**根据 coordinate.md 的"入口 C：style-write"触发矩阵：

| 触发条件 | 必须动作 |
|---------|---------|
| 🔴 冷启动 | 跳 cold_start.md 引导 |
| 主题相关正样本 < 3 | 提示"先 /style-feed 几篇相关主题" |
| 主题档案空白 | 提示"档案没有该主题样本，生成会偏通用" |
| 风格置信度 < 0.3 | 提示"档案不成熟，建议先 /style-feed 10 篇" |
| 草稿生成成功 | **强制**提示"/style-feedback 打分" |
| **生成偏离 > 50%（重试 2 次仍失败）** | 明确告知"档案需要补充"，绝不装作没事 |

**反馈末尾必须带**：
```
💡 下一步建议：
  - [A] /style-feedback 打分（强烈推荐，反向校准档案）
  - [B] 让 AI 重新生成（加调性要求）
  - [C] 手动修改后存档
```

**反馈里必须包含"风格匹配度报告"**（不能省）：
```
📊 风格匹配度：{N}% {🟢/🟡/🟠/🔴}
  7 维核对 + 样本还原 + 反样本规避 + 主题完成度
```

```
3️⃣ 平台（必填）
   朋友圈 / 微博 / 小红书 / 即刻 / 公众号

4️⃣ 调性（可选）
   克制 / 锋利 / 温柔 / 冷幽默 / 自嘲
   不指定就用档案默认
```
